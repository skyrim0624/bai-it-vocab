# 掰 it 生词本改造版

**英文太长？掰 it。看到生词？点一下加入生词本。**

这是基于 [掰 it v0.2.5](https://bai-it.app/) 的非官方个人学习改造版，新增了点词加入生词本、发音、读句测验和学习建议。原项目许可证为 [BSL 1.1](./LICENSE)，本仓库仅用于非商业学习分享，不是官方版本，也不要作为产品或服务分发。

## 下载安装

下载最新版：

[bai-it-v0.2.5-vocab.zip](https://github.com/skyrim0624/bai-it-vocab/releases/download/v0.2.5-vocab/bai-it-v0.2.5-vocab.zip)

Chrome 安装步骤：

1. 下载 zip 后解压。
2. 打开 Chrome，进入 `chrome://extensions/`。
3. 打开右上角“开发者模式”。
4. 点击“加载已解压的扩展程序”。
5. 选择解压出来的文件夹。

如果已经安装过旧版，重新构建或下载新版后，在 `chrome://extensions/` 里点“重新加载”，再刷新网页。

## 核心功能

### 自动拆英文长句

打开英文网页后，插件会把复杂长句拆成更容易阅读的结构块，保留缩进层级，帮助你看清主句、从句和修饰成分。

适合用在 X、Reddit、Medium、Substack、英文新闻、论文摘要等页面。

### 点词加入生词本

在网页正文里直接点击一个英文单词，插件会自动选中整个词，不需要拖动鼠标。

点词后会出现浮层：

- 中文释义
- 发音按钮
- 加入生词本
- 标记已掌握

加入生词本时会同时保存当前句子和来源 URL，方便之后用真实语境复习。

### 生词本

管理页新增“生词本”：

- 搜索单词、释义、原句、来源
- 按新词、学习中、已掌握筛选
- 按最近遇到、高频、A-Z 排序
- 展开查看语境句子
- 标记新词、学习中、已掌握
- 导出 Markdown / CSV

### 读句测验

生词本里有“读句测验”：

- 从你的生词里抽词
- 优先使用你真实读到过的语境句
- 可无限生成下一句
- 支持朗读句子

这个功能用于训练“看到词能不能在真实句子里读懂”，不是只背中文释义。

### 学习建议和 Codex 上下文

生词本会根据你的生词数量、掌握状态、出现频次和语境数量，生成一个粗略学习建议：

- 估计当前英语阅读水平区间
- 推荐优先复习词
- 推荐下一步学习动作
- 生成可复制给 Codex 的学习上下文

复制给 Codex 后，可以让 Codex 帮你继续做：

- 回顾生词
- 引申搭配
- 近义词辨析
- 制定 7 天学习计划
- 根据你的真实阅读材料评估下一步方向

## LLM 深度分析

不配置 API key 也可以使用本地拆句、生词标注、生词本和读句测验。

配置 LLM API key 后，可以解锁更完整的难句精析：

- 句式类型
- 为什么难读
- 表达学习点
- 语境化生词释义

支持的提供商：

| 提供商 | 默认模型 | 备注 |
| --- | --- | --- |
| Gemini | `gemini-3.1-flash-lite-preview` | 速度快，有免费额度 |
| ChatGPT | `gpt-4.1-mini` | 结构化输出稳定 |
| DeepSeek | `deepseek-chat` | 中文解释自然、成本低 |
| Qwen | `qwen3-flash` | 速度快 |
| Kimi | `kimi-k2.5` | 可选 |
| Codex 本机桥接 | `gpt-5.4-mini` | 个人私用，复用本机 Codex 登录态；点击翻译更快 |

### Codex 本机桥接

Codex Pro / Codex CLI 不是普通 OpenAI API Key。它使用本机 Codex 登录态，所以 Chrome 扩展不能、也不应该直接读取 `~/.codex/auth.json`。本项目提供一个只监听 `127.0.0.1` 的本机桥接服务：

```bash
npm run codex-bridge
```

然后在掰 it 管理页的设置中选择 `Codex`：

- Token：默认 `bait-local-codex`
- 模型：默认 `gpt-5.4-mini`
- 地址：扩展内置为 `http://127.0.0.1:17877`

注意：`gpt-5` 这个精确模型名当前不支持 ChatGPT 账号版 Codex；本项目默认使用更轻的 `gpt-5.4-mini`。如果你更看重深度分析质量，可以在设置页切回 `gpt-5.2` 或其他更强模型。

如果你的 Codex CLI 当前不支持默认模型，可以临时换模型启动：

```bash
BAIT_CODEX_MODEL=gpt-5.3-codex npm run codex-bridge
```

这个桥接适合个人本机使用，不适合公开部署给朋友共用。朋友要用 Codex Provider，需要他们在自己电脑上登录 Codex CLI 并启动自己的本机桥接服务。

## 隐私

- 没有后端服务器。
- 不需要注册账号。
- 学习数据保存在浏览器本地 IndexedDB。
- API key 只保存在本地浏览器里。
- LLM 请求由你的浏览器直接发往你配置的模型服务商。
- Codex Provider 只把请求发到 `127.0.0.1`，由本机桥接服务复用你自己的 Codex 登录态；扩展不会读取 Codex token。
